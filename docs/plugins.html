Plugin API

Markdown233 keeps the Obsidian-style mechanism: plugins register commands and panels, then users discover them through the command palette.

Minimum shape

export default {
  id: "example.plugin",
  name: "Example Plugin",
  activate(api) {
    api.registerCommand({
      id: "example.hello",
      title: () => "Hello",
      group: () => "Example",
      run: () => api.toast("Hello Markdown233")
    });
  }
}

Available API

registerCommand, registerPanel, getVaultNotes, getCurrentFile, switchPanel, and toast.

plugin.json

{
  "id": "example.plugin",
  "name": "Example Plugin",
  "version": "1.0.0",
  "permissions": ["vault:read", "commands:register"],
  "commands": [
    { "id": "hello", "title": "Hello", "group": "Example" }
  ],
  "panels": [
    { "id": "example.panel", "title": "Example" }
  ]
}
