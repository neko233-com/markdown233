Release, Signing, Updates

Windows and macOS releases should be signed, uploaded to GitHub Releases, and exposed through a Tauri updater manifest.

Commands

npm run e2e:screenshot
npm run release:windows
npm run release:signed:windows
OUT_DIR=release-check sh scripts/release-signed.sh

Required secrets

TAURI_SIGNING_PRIVATE_KEY and TAURI_SIGNING_PRIVATE_KEY_PASSWORD are required for updater signing. macOS also needs Apple signing and notarization credentials. Windows production builds should use an Authenticode certificate.
